# Security Policy

## Reporting a Vulnerability

If you find a vulnerability or accidentally exposed sensitive material, please open a private report through GitHub's security advisory flow if available, or contact the repository owner through the GitHub profile.

Do not open a public issue for secrets, credentials, or exploitable details.

## Scope

This repository is a static research artifact. It does not require runtime credentials and should not contain API keys, tokens, or private model logs. If future work adds live data collection, secrets must stay in local or platform-managed environment variables and must not be committed.
