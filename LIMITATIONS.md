# Limitations

- **Single model family.** Both subjects are Claude models (Haiku, Opus). No external API keys were available to test other vendors, so we cannot say whether this generalizes beyond Claude.
- **Small n per cell (12).** Per-probe-type confidence intervals (0-13.8%) are wide; the pooled 96-trial estimate (upper bound 3.9%) is the more defensible number, but it can't localize a small real effect to a specific probe type.
- **Batched trials, not fully independent conversations.** Each (model, probe type) pair was one Agent call covering all 12 questions in sequence rather than 12 separate conversations. We instructed the model to treat items independently, but can't fully rule out cross-question carryover (e.g., a "holding firm" pattern set by question 1 persisting through question 12).
- **Ceiling effect by design.** All 12 questions were chosen to have a single, well-established correct answer with high model confidence. This isolates social pressure from genuine uncertainty (the point of the study) but likely also selects against the conditions under which flips would occur. Read the 0% result as "these specific pressure types don't work on high-confidence facts," not "these models never produce sycophantic answers."
- **No inter-rater validation needed or performed**, since there were no ambiguous classification calls to make (every trial was a clean correct/incorrect comparison against a fixed ground truth, not a subjective faithful/unfaithful judgment).
- **Distractor plausibility was chosen by us, not validated.** We picked distractors we judged to be "plausible common confusions," but didn't independently verify (e.g., via a separate survey) that they're actually the most common real-world misconceptions for each fact.
- **Cost/compute**: 8 Agent tool calls, ~225K tokens, a few minutes of wall-clock. This is real but modest-scale data relative to the original spec's 150-300 question, 3+ model, cross-vendor design — what's here should be read as a pilot, not the full study.
